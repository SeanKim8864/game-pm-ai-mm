# 게임사업PM의 AI 활용 업무 — MM(성과 평가) 방향성 & 측정기준

## Context
게임사업PM이 AI를 활용한 업무를 성과 평가(KPI/MBO)에 포함시키려 한다.  
단순히 "AI를 썼다"가 아니라, **AI 활용이 만들어낸 비즈니스 임팩트**를 측정 가능한 형태로 정의해야 한다.  
범위: 데이터 분석/리포트 + PM 업무 전반.

---

## 1. 방향성: 2가지 축

### A. 효율성 (Efficiency) — "같은 일을 더 빠르게"

AI를 통해 기존 업무의 처리 속도를 높이고, 절약된 시간을 고부가가치 업무에 재투입하는 방향.

**게임사업PM 실무 예시:**

| 기존 업무 | AI 활용 방식 | Before → After |
|-----------|-------------|----------------|
| 주간 매출/DAU 리포트 작성 | AI가 DB에서 데이터 추출 → 자동 요약 → 드래프트 생성 | 매주 4시간 → 30분 (검토·보완만) |
| 업데이트 공지사항 작성 | 패치노트 원본 → AI가 유저 대상 공지 초안 생성 | 건당 2시간 → 20분 |
| 경쟁작 동향 조사 | AI가 앱스토어 순위·리뷰·뉴스 자동 수집·요약 | 주 3시간 → 30분 (큐레이션만) |
| 이벤트 성과 정리 | AI가 이벤트 전후 지표 비교 → 자동 성과 요약 | 이벤트당 3시간 → 40분 |
| 유관부서 미팅 회의록 정리 | AI 회의 녹음 → 자동 요약 + 액션아이템 추출 | 건당 1시간 → 10분 |

**핵심 평가 포인트:**
- 단순히 시간을 줄인 것이 아니라, **절약된 시간으로 무엇을 했는가**
- 예: 리포트 자동화로 주 10시간 절감 → 이 시간에 유저 세그먼트별 심층 분석을 새로 수행

---

### B. 품질 (Quality) — "같은 시간에 더 좋은 결과"

AI를 활용하여 분석의 깊이, 의사결정의 정확도, 산출물의 완성도를 높이는 방향.

**게임사업PM 실무 예시:**

| 업무 영역 | AI 활용 전 | AI 활용 후 |
|-----------|-----------|-----------|
| 매출 분석 | 전체 매출 추이 확인 수준 | AI가 결제 패턴·유저 코호트별 ARPU 변화·이상 징후까지 자동 분석, PM은 인사이트 해석과 시책 수립에 집중 |
| BM 가격 책정 | 경험 기반 + 단순 벤치마크 | AI 가격 시뮬레이션으로 탄력성 분석 → 가격대별 예상 매출·전환율 시나리오 3개 비교 후 결정 |
| 업데이트 기획 검토 | 과거 유사 업데이트 1~2건 수동 비교 | AI가 과거 전체 업데이트 이력의 지표 변화를 패턴 분석 → "이 유형 업데이트는 평균 DAU +5%, 7일 리텐션 -2% 영향" 형태로 예측 제공 |
| 유저 VOC 분석 | CS팀에서 올라온 주요 이슈 요약 읽기 | AI가 커뮤니티·앱리뷰·CS 데이터를 통합 감성 분석 → 이슈별 심각도·영향 유저 규모·트렌드 시각화 |
| 마케팅 소재 검토 | PM 개인 감각 + 과거 경험 | AI가 과거 소재별 CTR/CPI 데이터 분석 → "이 유형 소재는 타겟 연령층에서 CTR 평균 1.8배" 데이터 기반 피드백 |

**핵심 평가 포인트:**
- 의사결정 시 **"감"이 아닌 "데이터 기반 근거"가 포함되었는가**
- AI 분석 결과를 PM이 해석하여 **실행 가능한 시책으로 전환했는가**
- 이해관계자(개발팀, 경영진)에게 **더 설득력 있는 보고가 가능해졌는가**

## 3. 평가 등급 기준 (안)

### S 등급 (탁월) — "AI로 비즈니스를 바꿨다"
- 측정 가능한 매출/KPI 개선에 AI 분석이 직접 기여 (증빙 가능)
- 팀/조직 차원의 AI 역량 확산을 주도 (가이드, 교육, 표준 프로세스 수립)
- **구체적 시나리오**: PM이 AI 분석으로 코호트별 ARPU 하락 원인을 발견 → BM 가격 조정 시책 실행 → 월 매출 +15% 개선. 이 분석 프로세스를 표준화하여 팀 전체에 전파.

### A 등급 (우수) — "AI가 업무의 일부가 되었다"
- 정기 업무 3건 이상에 AI를 체계적으로 통합
- 시간 절감률 50% 이상인 업무가 2건 이상
- AI 기반 분석이 의사결정에 반영된 사례 분기 2건 이상
- **구체적 시나리오**: 주간 리포트·경쟁작 분석·이벤트 성과 정리를 모두 AI로 자동화. AI 가격 시뮬레이션을 활용해 시즌 BM 가격을 데이터 기반으로 결정, 목표 매출 달성.

### B 등급 (보통) — "AI를 쓰기 시작했다"
- AI를 간헐적으로 활용하여 일부 업무에서 효율 개선 확인
- 시간 절감 효과는 있으나 체계적 프로세스로 정착되지 않음
- **구체적 시나리오**: ChatGPT로 공지사항 초안을 가끔 작성하거나, 데이터 요약을 요청하는 수준. 특정 업무에서 편리함은 느끼지만 정기적으로 활용하진 않음.

### C 등급 (미흡) — "시도는 했으나 성과 없음"
- AI 도구 사용 이력은 있으나 업무 성과와 연결된 사례가 없음
- AI 산출물을 그대로 사용하여 오류·품질 이슈 발생
- **구체적 시나리오**: AI를 몇 번 써봤으나 프롬프트 작성이 어려워 포기, 또는 AI가 생성한 부정확한 데이터를 검증 없이 보고하여 신뢰도 이슈 발생.

## 4. MM 반영 시 유의사항

### 4-1. "AI 사용 여부"가 아닌 "AI 활용 성과"를 평가
- **잘못된 예**: "ChatGPT를 주 5회 사용했으므로 A등급"
- **올바른 예**: "ChatGPT를 활용해 경쟁작 분석 시간을 70% 줄이고, 절약된 시간으로 자사 게임의 유저 이탈 패턴 심층 분석을 새로 수행하여 리텐션 시책에 반영"
- AI 도구 사용 횟수·빈도 자체는 평가 대상이 아님

### 4-2. Before/After 비교 가능한 구조 설계
- 평가 시작 전에 **현재 업무별 소요시간·산출물 수·품질 수준을 베이스라인으로 기록**
- 예시 베이스라인 기록 양식:

| 업무명 | 주기 | 현재 소요시간 | 현재 산출물 수준 | AI 활용 계획 |
|--------|------|-------------|-----------------|-------------|
| 주간 매출 리포트 | 주 1회 | 4시간 | 전체 매출 추이 + 주요 지표 3개 | AI 자동 요약 + 코호트별 분석 추가 |
| 경쟁작 동향 보고 | 주 1회 | 3시간 | 상위 5개 경쟁작 순위 변동 | AI 자동 수집 + 리뷰 감성분석 추가 |
| 이벤트 성과 분석 | 이벤트당 | 3시간 | 참여율·매출 변화 요약 | AI 자동 비교 + 유사 이벤트 패턴 분석 |

### 4-3. 업무 포트폴리오 관점으로 평가
- AI 절감 시간의 **재투자 내역**을 함께 기록
- 예시:
  - 절감: 주간 리포트 3h + 공지 작성 1.5h + 경쟁작 조사 2.5h = **주 7시간 절감**
  - 재투자: 유저 세그먼트별 LTV 분석 (3h) + BM 가격 시뮬레이션 (2h) + 개발팀 협업 미팅 확대 (2h)
- "AI로 시간을 벌었다" + "그 시간을 이렇게 썼다" = 완전한 성과 스토리

### 4-4. 과도한 AI 의존 리스크 관리
- **검증 체크리스트 의무화** 예시:
  - [ ] AI 산출물의 수치가 원본 데이터와 일치하는가?
  - [ ] AI의 해석/요약에 누락된 맥락은 없는가?
  - [ ] AI가 제시한 인사이트가 도메인 지식과 부합하는가?
  - [ ] 할루시네이션(없는 데이터 생성) 여부를 확인했는가?
- 오류 발견 시 **원인 분석 + 프롬프트 개선**까지 이어지는 피드백 루프 운영 여부도 평가

### 4-5. 팀/조직 기여 가중치
- 개인 성과 70% + 조직 기여 30% (비율은 조직 상황에 따라 조정)
- 조직 기여 인정 항목:
  - AI 활용 가이드/프롬프트 템플릿 공유
  - 팀 내 AI 활용 교육·워크숍 진행
  - 타 부서 AI 도입 컨설팅 지원
  - AI 활용 실패 사례 공유 (실패에서 배운 점도 가치 있음)

## 5. 실행 로드맵 (제안)

### Q1: 기반 구축
- [ ] PM 업무 목록 작성 및 AI 활용 가능 업무 식별
- [ ] 업무별 현재 소요시간·산출물 수준 베이스라인 측정
- [ ] AI 도구 선정 및 접근 권한 확보
- [ ] 측정 프레임워크 초안 합의

### Q2: 파일럿 적용
- [ ] 핵심 업무 2~3개에 AI 우선 적용 (예: 주간 리포트, 경쟁작 분석)
- [ ] 주단위 효과 측정 시작 (시간 절감, 품질 변화)
- [ ] AI 활용 실패/성공 사례 기록 시작
- [ ] 프롬프트 라이브러리 구축 시작

### Q3: 확대 및 중간 평가
- [ ] 파일럿 결과 기반으로 적용 업무 확대
- [ ] 중간 성과 평가 실시 (등급 기준 시범 적용)
- [ ] 팀 내 공유 세션 진행 (성공/실패 사례)
- [ ] 측정 프레임워크 보완

### Q4: 정식 평가 및 차기 계획
- [ ] AI 활용 성과 정식 평가 (MM 반영)
- [ ] 등급별 피드백 및 개선 계획 수립
- [ ] 차기 분기/연도 AI 활용 목표 설정
- [ ] 우수 사례 전사 공유

## 요약

> **방향**: 효율성·품질 2축으로 AI 활용 가치를 분류  
> **측정**: 정량(시간절감, 산출물, 비즈니스 기여) + 정성(의사결정, 확산, 창의성) 병행  
> **등급**: S/A/B/C 4단계, 각 등급에 구체적 시나리오 기준 명시  
> **핵심 원칙**: "AI를 썼는가"가 아니라 "AI로 무엇을 만들었는가"  
> **성과 스토리**: "AI로 시간을 절약했다" + "그 시간으로 이것을 했다" = 완전한 평가 근거
